entra-auth-app-sample

Microsoft Entra ID 등록 가이드 — /api/message 호출 성공까지

lgup-m365-mcp 적용 시 (2026-07) — 아래 절차는 범용 FastAPI+MSAL 샘플(포트 8000, /api/message) 기준입니다. lgup-m365-mcp에 적용할 때는 포트 8080, 보호 API는 /mcp, 브라우저 테스트 UI는 /auth-ui·/drm-ui이며 SPA Redirect URI로 http://localhost:8080/auth-ui, http://localhost:8080/drm-ui(및 배포 도메인)를 등록하세요. 노출 scope는 access_as_user, Application ID URI는 api://<clientId>입니다.

FastAPI 백엔드 + MSAL Browser 프론트엔드 샘플에서 로그인과 보호 API 호출이 성공할 때까지 거친 앱 등록 과정을 단계별로 정리합니다. 실제로 마주친 오류 (AADSTS500011, AADSTS9002326)와 해결책도 함께 담았습니다.

전체 순서도

시작/종료 작업 단계 판단(오류 분기) 성공

단계별 상세 가이드

1

앱 등록(App registration) 생성

Azure Portal → Microsoft Entra ID → App registrations → New registration.

항목값
Nameentra-auth-app-sample (자유)
Supported account types단일 테넌트(권장)
Redirect URI이 단계에서는 비워두고 4단계에서 SPA로 추가

생성 후 Overview에서 Directory(tenant) ID와 Application(client) ID를 복사합니다.

2

.env 작성 (최소 2개)

나머지 두 변수는 ENTRA_CLIENT_ID로부터 자동 파생되므로 표준 설정이면 두 값만 필수입니다.

ENTRA_TENANT_ID=<directory-tenant-id>
ENTRA_CLIENT_ID=<application-client-id>
# 선택(표준이면 생략 가능)
ENTRA_AUDIENCE=api://<application-client-id>
ENTRA_API_SCOPE=api://<application-client-id>/access_as_user
주의 ENTRA_AUDIENCE/ENTRA_API_SCOPE를 적을 거면 플레이스홀더(<application-client-id>)를 실제 client ID로 반드시 교체하세요. 특히 ENTRA_API_SCOPE는 프론트엔드가 토큰을 요청할 때 그대로 쓰여 잘못된 값이면 로그인이 실패합니다.
3

Expose an API — App ID URI + 스코프

App registration → Expose an API:

  1. Application ID URI → Add → 기본값 api://<client-id> 저장
  2. + Add a scope → 이름 access_as_user, Who can consent: Admins and users, State: Enabled
이 단계를 건너뛰면 → AADSTS500011 "resource principal named api://… was not found" — App ID URI/스코프가 없어 리소스를 못 찾는 경우입니다.
4

Authentication — 리디렉션 URI를 SPA로

App registration → Authentication → + Add a platform → Single-page application 선택 후 입력:

http://localhost:8000
Web 플랫폼으로 넣으면 → AADSTS9002326 "Cross-origin token redemption is permitted only for the 'Single-Page Application' client-type." MSAL Browser는 PKCE+CORS라 반드시 SPA 플랫폼이어야 합니다.
origin 정확히 일치 localhost와 127.0.0.1은 서로 다른 origin입니다. 접속 주소와 동일하게 등록하세요. 끝에 슬래시/경로 없이 http://localhost:8000.
5

백엔드 실행 후 접속

uv run uvicorn app.main:app --reload
# 또는
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

브라우저에서 http://localhost:8000 접속. 하단 상태가 "Not signed in." 이면 설정 인식 성공.

.env 변경 후엔 서버 재시작 설정은 시작 시 1회 로드됩니다. 단, 앱 등록(Portal) 변경은 즉시 반영되어 서버 재시작이 필요 없습니다.
6

로그인 → Post Message

  1. Authentication Button 클릭 → Microsoft 로그인 + 최초 1회 동의
  2. 상태가 "Signed in as <email>" 로 변경되면 성공
  3. Post Message 클릭 → 응답 패널에 200 응답(JSON) 표시
성공 응답 예시 
{
  "authenticated": true,
  "request": { "message": "..." },
  "user": { "id": "...", "email": "...", "name": "...", "tenantId": "...", "scopes": "access_as_user" },
  "token": { "issuer": "...", "audience": "...", "expiresAt": ... },
  "claims": { /* 디코딩된 전체 JWT 클레임 */ }
}

마주친 오류 & 해결 요약

오류 코드 메시지 요지 원인 해결
AADSTS500011 resource principal named api://… was not found App ID URI/스코프 미설정 3단계: Expose an API에서 App ID URI 추가 + access_as_user 노출
AADSTS9002326 Cross-origin token redemption … only for 'Single-Page Application' 리디렉션 URI가 Web 플랫폼 4단계: 해당 URI를 SPA 플랫폼으로 이동
AADSTS50011 redirect URI mismatch 등록 origin ≠ 접속 origin 접속 주소와 동일한 origin 등록 (localhost vs 127.0.0.1)
401 Invalid or expired access token audience/issuer 불일치 서버 로그 Entra token validation failed의 expected vs actual 비교. aud가 bare ID면 ENTRA_AUDIENCE에 콤마로 추가
최소 필수 체크리스트
  1. App registration 생성 → tenant/client ID 확보
  2. .env에 ENTRA_TENANT_ID, ENTRA_CLIENT_ID
  3. Expose an API: App ID URI + access_as_user
  4. Authentication: SPA 플랫폼에 http://localhost:8000
  5. localhost로 접속 → 로그인 → Post Message → 200
